# GemiGo 产品概念：工具版小红书 + 小程序运行时

## 核心公式

```
GemiGo = 小红书（社区分享） + 小程序（运行时） + 桌面本地能力
```

---

## 一句话定位

> **"分享你的工具，使用别人的工具"**

## 两种用户角色

### 创作者（分享工具的人）

```
我用 Cursor 写了一个 Markdown 编辑器
          ↓
    一键部署到 GemiGo
          ↓
    填写介绍，打标签
          ↓
    进入社区被发现
          ↓
    获得点赞、收藏、评论
          ↓
    有人在用我的工具！🎉
```

### 使用者（发现工具的人）

```
我需要一个 AI 帮我写周报
          ↓
    打开 GemiGo 搜索 "AI 周报"
          ↓
    看到几个工具，挑评分最高的
          ↓
    点击直接使用（无需下载安装）
          ↓
    好用！点赞收藏
          ↓
    下次直接从收藏打开
```

## 平台能力

### Web 端

| 能力 | 描述 |
|------|------|
| 发现 Feed | 基于兴趣推荐工具 |
| 搜索 | 按功能、标签搜索 |
| 运行 | 点击即用，iframe 内运行 |
| 社交 | 点赞、收藏、评论、关注 |

### 桌面端（额外能力）

| 能力 | 描述 |
|------|------|
| 本地文件 | 工具可读写本地文件 |
| 系统集成 | 全局快捷键、托盘常驻 |
| 离线使用 | 缓存常用工具 |
| 跨工具操作 | 工具间共享数据 |

## 与小红书类比

| 小红书 | GemiGo |
|--------|--------|
| 分享生活经验 | 分享实用工具 |
| 图文/视频笔记 | 可运行的 Web 应用 |
| 发现好物 | 发现好工具 |
| 种草 → 购买 | 发现 → 使用 |
| 博主 | 工具创作者 |

---

## 与小程序类比

| 微信小程序 | GemiGo |
|-----------|--------|
| 微信生态内运行 | GemiGo 内运行 |
| 扫码进入 | 搜索/点击进入 |
| 无需下载安装 | 无需下载安装 |
| 调用微信能力 | 调用 GemiGo 能力（AI、存储...） |
| 🔒 封闭在微信 | ✅ 开放的 Web |

---

## 差异化优势

| 对比 | GemiGo 优势 |
|------|------------|
| vs 小程序 | 开放生态，不绑定单一平台 |
| vs GitHub | 社区化发现，不只是代码托管 |
| vs Product Hunt | 可直接运行，不只是导流 |
| vs 应用商店 | Web 技术，开发门槛极低 |

## 社区运营亮点

### 让创作者有动力

- 🏆 排行榜：周榜 / 月榜 / 总榜
- ❤️ 点赞数显示
- 💬 用户评论反馈
- 📊 使用数据面板

### 让用户愿意发现

- 🎯 智能推荐（基于使用习惯）
- 🏷️ 标签分类（效率、AI、设计...）
- ⭐ 精选专题（本周最佳、新人必备）
- 👥 关注喜欢的创作者

---

## 实施路径

### Phase 1: 社区 MVP

- 升级现有应用市场 → 社区化
- 添加点赞、收藏、评论
- 创作者个人主页
- 发现 Feed

### Phase 2: 桌面客户端

- 封装 Web 版为桌面应用
- 添加本地能力（文件系统）
- 全局快捷键唤醒

### Phase 3: 能力开放

- Plugin SDK 发布
- 创作者可调用平台能力
- 工具间通信
